## 📝 Описание проекта

**WeatherBot** — это специализированный и функциональный бот для платформы Telegram, предназначенный для предоставления пользователям актуальной информации о погоде в заданных локациях. Проект разработан с использованием архитектурного подхода, разделяющего логику работы бота, взаимодействие с внешними API (погодой) и управление пользовательскими данными.

Бот позволяет получать подробные прогнозы погоды, интегрируя современные возможности Telegram Bot API и обеспечивая надежное хранение данных пользователей.

## ✨ Ключевые особенности

*   **Интеграция с Telegram:** Полное использование Telegram Bot API для взаимодействия с пользователем в удобном мессенджере.
*   **Актуальные данные о погоде:** Получение профессионально структурированных погодных прогнозов через внешние REST API (требуется конфигурация ключей).
*   **Постоянное хранение данных:** Пользовательские данные и настройки сохраняются локально в файле `User_Data.json`, обеспечивая персонализацию опыта использования.
*   **Модульная архитектура:** Разделение логики на компоненты (`weather_app.py`, `storage.py`, `bot.py`) повышает читаемость, тестируемость и масштабируемость кода.

## 🛠️ Технологический стек (Tech Stack)

*   **Язык программирования:** Python
*   **Библиотеки для API-взаимодействия:** `requests` (Для запросов к погодным API).
*   **Telegram Боты:** `pyTelegramBotAPI` (Основная библиотека для управления ботом).
*   **Управление конфигурацией:** `python-dotenv` (Используется для безопасного хранения ключей API и токенов бота).

## 🚀 Установка и Настройка

### Шаг 1: Клонирование репозитория

Клонируйте проект на вашу локальную машину:

```bash
git clone [ВАШ_РЕПОЗИТОРИЙ]
cd API_weather
```

### Шаг 2: Создание виртуального окружения

Всегда рекомендуется использовать виртуальное окружение для изоляции зависимостей проекта:

```bash
python3 -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate    # Для Windows
```

### Шаг 3: Установка зависимостей

Установите все необходимые библиотеки из файла `requirements.txt`:

```bash
pip install -r requirements.txt
```

### Шаг 4: Настройка переменных окружения (Важно!)

Перед первым запуском вам необходимо создать файл `.env` в корневой директории проекта (`API_weather/`). В этот файл должны быть добавлены все секретные ключи, такие как токен вашего Telegram бота и ключ API для погоды.

**Пример файла `.env`:**
```ini
# Токен вашего Telegram бота (полученный от @BotFather)
TELEGRAM_BOT_TOKEN=ВАШ_ТОКЕН_ТЕЛЕГРАМА

# Ключ доступа к внешнему API погоды
WEATHER_API_KEY=ВАШ_ПОГОДНЫЙ_API_КЛЮЧ
```
*(Используйте `python-dotenv` для автоматического загружения этих переменных в окружение.)*

## ▶️ Запуск проекта

После завершения настройки убедитесь, что ваше виртуальное окружение активировано. Для запуска бота выполните команду:

```bash
python bot.py
```

Бот запустится и начнет слушать входящие сообщения в Telegram. Вся оперативная информация о работе будет логироваться в файл `weather_bot.log`.

### Структура проекта

| Файл | Назначение |
| :--- | :--- |
| `bot.py` | Главная точка входа, инициализирует и запускает бота. Обрабатывает входящие сообщения. |
| `weather_app.py` | Ядро бизнес-логики. Отвечает за взаимодействие с внешним погодным API (парсинг данных). |
| `storage.py` | Модуль управления данными. Реализует логику сохранения и загрузки пользовательских настроек (`User_Data.json`). |
| `User_Data.json` | Файл хранения данных пользователей (хранилище состояния бота). |
| `requirements.txt` | Список всех внешних библиотек, необходимых для работы проекта. |

## 📚 Разработка и расширение

Проект спроектирован модульно, что позволяет легко добавлять новые функции:

1.  **Улучшение API-интеграции:** Для добавления данных о качестве воздуха (AQI) или прогнозах на неделю достаточно изменить логику в `weather_app.py` и обновить ключи в `.env`.
2.  **Добавление команд:** Новые команды Telegram могут быть реализованы путем расширения хэндлеров в файле `bot.py`.
